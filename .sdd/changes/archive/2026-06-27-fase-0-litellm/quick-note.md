# Quick Note — fase-0-litellm

> SDD-fast change. Reduced trace for small scope.
> Generated by `sdd-fast` on 2026-06-27.

## What

Agregar el servicio LiteLLM al `docker-compose.yaml` existente y crear `litellm_config.yaml` en la raíz del repo. LiteLLM actuará como proxy OpenAI-compatible que enruta las peticiones al Ollama que ya corre en Docker, exponiendo el modelo `llama3.2:3b` bajo el alias `llama3` en el puerto `4000`.

## Why

Los agentes Java que se construirán a futuro deben pegarle a un endpoint estilo OpenAI (`/v1/chat/completions`) y no directamente a Ollama. Sin LiteLLM no hay proxy, y sin proxy ningún agente puede arrancar. Esta es la pieza de infraestructura que desbloquea toda la Fase 1 en adelante.

## Files in scope

- `docker-compose.yaml` — agregar el servicio `litellm` con imagen `ghcr.io/berriai/litellm:main-latest`, puerto `4000:4000`, volumen del config, y `depends_on: ollama`
- `litellm_config.yaml` — archivo de config nuevo en la raíz del repo; declara el modelo `llama3` apuntando a `ollama/llama3.2:3b` con `api_base: http://ollama:11434`

## Acceptance

El siguiente `curl` ejecutado desde el host debe devolver una respuesta JSON con un campo `choices[0].message.content` no vacío, con HTTP 200:

```bash
curl -s -X POST http://localhost:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3",
    "messages": [{"role": "user", "content": "decime hola en una palabra"}]
  }' | jq '.choices[0].message.content'
```

## Out of scope

- No se modifica ningún otro servicio del compose (ollama, qdrant, flowise, langfuse, langfuse-db).
- No se configura autenticación en LiteLLM (queda sin `master_key` por ahora — entorno local).
- No se configura integración con Langfuse (callbacks de observabilidad).
- No se agregan modelos cloud ni embeddings — solo `llama3.2:3b` vía Ollama.
- No se crean agentes Java en este cambio.
- No se toca `.env` — LiteLLM en modo local sin secrets.

## Validation plan

1. Levantar (o reiniciar) el stack:

```bash
docker compose up -d litellm
```

2. Esperar que el contenedor esté healthy (~10s) y verificar logs:

```bash
docker logs litellm --tail 20
```

3. Smoke test OpenAI-compatible:

```bash
curl -s -X POST http://localhost:4000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3",
    "messages": [{"role": "user", "content": "decime hola en una palabra"}]
  }' | jq '.choices[0].message.content'
```

Resultado esperado: string no vacío (ej. `"¡Hola!"`). HTTP 200 en la respuesta.

4. Verificar que Ollama sigue respondiendo (regresión):

```bash
curl -s http://localhost:11434/api/tags | jq '.models[].name'
```
